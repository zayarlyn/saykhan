# Saykhan — Clinic Inventory & Patient Session Management

**Date:** 2026-04-06
**Status:** Approved

---

## Overview

Saykhan is a web application for clinic owners to manage inventory, patient sessions, income, and expenses in one place. The primary user is the clinic owner operating solo.

---

## Tech Stack

| Layer | Choice |
|---|---|
| Framework | Next.js 14 (App Router) + TypeScript |
| ORM | Prisma |
| Database | PostgreSQL |
| UI | shadcn/ui + Tailwind CSS |
| Charts | Recharts |
| Auth | Single-user password login (no OAuth) |
| Deployment | Vercel (app) + Railway or Supabase (Postgres) |

## Data Model

### Lookup / Config Tables (user-managed)

- **MedicationCategory** — `id`, `name`
- **ServiceType** — `id`, `name` (20+ clinic-defined service types, e.g. Stitching, Consultation)
- **PaymentMethod** — `id`, `name` (e.g. Cash, Bank Transfer)
- **ExpenseCategory** — `id`, `name` (e.g. Utilities, Salary)

### Core Tables

**Patient**
- `id`, `name` (required), `createdAt`

**Medication** (inventory item)
- `id`, `name`, `categoryId`, `cost`, `sellingPrice`, `stock` (integer, kept in sync), `threshold`, `createdAt`
- `status` (in-stock / out-of-stock) — derived from `stock > 0`
- `nearestExpiryDate` — derived from earliest-expiring active RestockBatchItem

**RestockBatch** (restock transaction)
- `id`, `date`, `createdAt`
- Auto-generates one **Expense** record on creation (totalCost = sum of all items)

**RestockBatchItem**
- `id`, `restockBatchId`, `medicationId`, `quantity`, `costPerUnit`, `expiryDate`
- On create: `medication.stock += quantity`

**PatientSession** (income record)
- `id`, `patientId`, `serviceTypeId`, `paymentMethodId`, `date`, `description`, `paymentAmount`, `createdAt`

**SessionMedication** (medications used in a session)
- `id`, `sessionId`, `medicationId`, `quantity`, `unitCost`, `sellingPrice`
- On create: `medication.stock -= quantity` (rejected if stock would go negative)

**Expense**
- `id`, `categoryId` (nullable — not required for RESTOCK type), `type` (RESTOCK | MANUAL), `amount`, `description`, `date`, `createdAt`
- RESTOCK expenses are auto-generated by RestockBatch; MANUAL expenses are user-created

## Pages & Features

### Dashboard
- Low-stock alerts: medications where `stock ≤ threshold`
- Current month financial summary:
  - **Revenue** = sum of `patientSession.paymentAmount`
  - **Inventory Cost** = sum of `sessionMedication.quantity × unitCost`
  - **Adjusted Expenses** = sum of `expense.amount` where `type = MANUAL`
  - **Net Profit** = Revenue − Inventory Cost − Adjusted Expenses
- Restock (RESTOCK type) expenses are excluded from Adjusted Expenses and dashboard report

### Inventory
- Medication list: name, category, cost, selling price, stock, status, nearest expiry, threshold
- Add / edit medication
- Restock: create a RestockBatch with multiple line items — auto-increments stock and creates expense
- Threshold is editable per medication; low-stock badge shown when `stock ≤ threshold`

### Patient Sessions (Income)
- Session list, filterable by date / patient / service type
- Create session: patient (select existing or create new inline), service type, date/time, description, payment amount/method, medications used (select med + quantity — stock deducted on save)
- All session saves are transactional: stock deductions and session record committed together or rolled back

### Patient List
- Searchable list of patients by name
- Patient profile: name + full session history

### Expenses
- List: category, amount, date, description
- Add / edit manual expenses with user-defined categories
- Restock-generated expenses visible but not manually editable

### Settings
- Manage lookup tables: ServiceType, PaymentMethod, MedicationCategory, ExpenseCategory

## Business Logic

### Stock Sync
- All stock mutations (restock increments, session decrements) happen inside a database transaction
- If a session would cause any medication's stock to go below 0, the entire session is rejected

### Dashboard Calculations
- Computed server-side via Prisma aggregates scoped to the current calendar month
- Restock expenses (type = RESTOCK) are excluded from Adjusted Expenses

### Low-Stock Alerts
- Queried on dashboard load: `WHERE stock <= threshold`
- Threshold is set per medication and editable from inventory list inline

---

## Error Handling & Validation

- Client-side: react-hook-form + zod schemas
- Server-side: API route validation mirrors zod schemas; transactions roll back on any failure
- Negative stock prevention enforced at the database transaction level
