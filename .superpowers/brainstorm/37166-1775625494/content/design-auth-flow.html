Design: Auth Flow

How login works with Supabase magic link + @supabase/ssr

Step 1 — Admin visits /login
Login Page
Enter your email to receive a login link
admin@clinic.com
Send Magic Link
→ Calls supabase.auth.signInWithOtp({ email })
↓
Step 2 — Supabase sends email

Page shows "Check your email" confirmation. No password ever stored or transmitted.

↓
Step 3 — Admin clicks link → /auth/callback?code=...

New route handler calls supabase.auth.exchangeCodeForSession(code). Session written to cookie. Redirect to /dashboard.

↓
Step 4 — proxy.ts guards all routes

On every request, the proxy creates a Supabase server client and calls supabase.auth.getUser(). If no valid session → redirect to /login or 401. Replaces iron-session check entirely.
