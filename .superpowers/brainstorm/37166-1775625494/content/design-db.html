Design: Database Migration

Moving from your current Postgres to Supabase Postgres

Step 1 — Create Supabase project

Create a new project at supabase.com. Get the Postgres connection string from Project Settings → Database → Connection string (URI). Enable email auth + magic links under Authentication → Providers. Create the admin user under Authentication → Users.

↓
Step 2 — Run Prisma migrations on Supabase DB

Point DATABASE_URL at the Supabase connection string and run:

npx prisma migrate deploy

This applies all existing migrations to the fresh Supabase database — no schema rewrite needed.

↓
Step 3 — Migrate existing data (if needed)

If you have production data to keep, dump it from your current DB and restore it to Supabase:

pg_dump $OLD_DATABASE_URL | psql $SUPABASE_DATABASE_URL

Skip this step if you're starting fresh or using the demo seed.

↓
Step 4 — Update env vars in Vercel

Replace old vars with:

  • DATABASE_URL → Supabase Postgres URI
  • NEXT_PUBLIC_SUPABASE_URL → from Supabase dashboard
  • NEXT_PUBLIC_SUPABASE_ANON_KEY → from Supabase dashboard
